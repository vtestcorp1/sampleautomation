LESS directory structure
All LESS files with common style patterns and mixins reside under blink/app/resources/css. Rest of the module related LESS files is created in the same directory as its HTML and JS files.
LESS class naming
Every class name added in a LESS file should have a prefix 'bk'.
.bk-test-class-name {
//CSS rules
}
How to structure a LESS class?
We have base patterns defined in various less file which you can find inside css folder. They should be self-explanatory. Whenever we create a new module we should try to re-use those patterns to create more patterns which will help in bringing visual harmony across the app.
Note:
- Maintain the order of elements in html as they appear visually in design
- !important: important overrides and other css rules defined for a particular element. We never want to use it.
Example:
.bk-filter-list-heading {
/* Positioning */
.pa;
top: 0;
right: 0
z-index: 10;
/* Display & Box Model */
.dib;
width: 100px;
height: 100px;
.bk-padding-all-10;>
.bk-margin-vertical-5;
/* Color */
.bg-grey2;
.color-grey8;
/* Text */
.font-family-semibold;
.text-size-larger;
.text-uppercase;
.bk-align-right;
}
